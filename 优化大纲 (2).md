## 二、优化阶段规划

### Phase 1：打通 Agent 运行流程（核心）
> 🔴 紧急 | 2-3 天
>
> **目标：点击"运行"按钮后，Agent 真正执行并显示结果**

1. **补充 `langgraph`/`langchain-core` 到 pyproject.toml**（否则无法运行）
2. **修复 `nodes.py` 异步调用** — 将节点函数改为 `async def`，或修复事件循环处理
3. **修复 GraphCompiler 条件边编译** — 按 source 分组条件边，每 source 只调用一次 `add_conditional_edges`
4. **修复 TRPG 模板 graph.json** — 移除与条件边冲突的普通边
5. **`_on_run_agent` 使用项目编译的图** — 调用 `set_graph(project_manager.compile_graph())`
6. 初始化 Feature 系统注册

📄 **详见：[优化步骤P1-打通Agent运行流程.md](优化步骤P1-打通Agent运行流程.md)**

---

### Phase 2：编辑器体验修复
> 🟠 高 | 2-3 天
>
> **目标：编辑 → 保存 → 加载 完整闭环**

1. Prompt 编辑器持久化到文件
2. 图编辑器 `_delete()` 调用 `remove_node()` 修复边残留
3. 图编辑器拖拽连线交互
4. 节点 ID 自动生成（避免冲突）
5. 清理调试 print 语句

📄 **详见：[优化步骤P2-编辑器体验修复.md](优化步骤P2-编辑器体验修复.md)**

---

### Phase 3：工具与 Feature 打通
> 🟡 中 | 3-4 天
>
> **目标：Agent 的命令能真正改变游戏状态**

1. 统一工具管理器与 `tools.py` 的工具定义
2. `tools.py` 工具连接到 Feature 系统/Repository
3. 工具测试面板真实调用
4. Feature 系统事件订阅完善

📄 **详见：[优化步骤P3-工具与Feature打通.md](优化步骤P3-工具与Feature打通.md)**
