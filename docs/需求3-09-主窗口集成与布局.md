## 1. 集成方案

### 1.1 布局变更

当前主窗口是三栏布局（LeftPanel + CenterPanel），AI 助手面板有两种集成方式：

**方案 A（推荐）：作为 CenterPanel 的标签页**

```
CenterPanel (QTabWidget):
  ├── [图查看] GraphEditorWidget
  ├── [提示词] PromptEditorWidget
  ├── [Skill] SkillManagerWidget
  ├── [🤖 AI 助手] AIAssistantPanel    ← 新增标签页
  ├── [运行] RuntimePanel
  ├── [事件] EventMonitor
  └── [日志] LogViewer
```

**优点**：与现有标签页风格一致，用户切换方便，不改变整体布局
**缺点**：AI 助手面板宽度受限于中央区域

**方案 B：右侧面板**

```
┌──────────┬──────────────────────┬───────────────┐
│ LeftPanel│    CenterPanel       │ AIAssistant   │
│          │                      │ Panel         │
│          │                      │ (可折叠)      │
└──────────┴──────────────────────┴───────────────┘
```

**优点**：AI 助手始终可见，可以同时查看编辑器和 AI
**缺点**：需要修改主布局，占用屏幕空间

**选择方案 A**，因为：
1. 改动最小，只需在 CenterPanel 中添加一个标签页
2. 与现有标签页风格一致
3. AI 助手不需要始终可见（用户主动切换即可）

### 1.2 菜单入口

在 **Agent(&A)** 菜单中添加 AI 助手入口：

```
Agent(&A):
  ├── 运行 Agent          F5
  ├── 停止               Shift+F5
  ├── ───分隔符───
  ├── 🤖 AI 助手         Ctrl+Shift+A    ← 新增
```

### 2.3 添加菜单处理方法

在 `MainWindow` 类中添加方法：

```python
def _on_open_ai_assistant(self):
    """切换到 AI 助手标签页"""
    # 获取 CenterPanel 实例
    center_panel = self._center_panel  # 根据实际属性名调整

    # 找到 AI 助手标签页的索引
    for i in range(center_panel._tab_widget.count()):
        if center_panel._tab_widget.tabText(i) == "🤖 AI 助手":
            center_panel._tab_widget.setCurrentIndex(i)
            return

    # 如果标签页不存在（项目未打开），显示提示
    from foundation.logger import get_logger
    logger = get_logger(__name__)
    logger.warning("AI 助手标签页不存在，请先打开项目")
```

## 5. 注意事项

1. **属性名适配**：上面的代码中 `self._center_panel` 需要根据实际的主窗口代码调整属性名。通过阅读 `MainWindow._setup_ui()` 找到 CenterPanel 的实际变量名
2. **懒加载**：如果其他标签页使用了懒加载模式，AI 助手也可以采用同样的模式。但由于 AIAssistantPanel 本身很轻量（没有复杂的初始化），直接创建即可
3. **标签页图标**：如果项目使用了自定义标签页图标，AI 助手标签页也应该添加对应的图标
4. **主题适配**：AIAssistantPanel 的样式需要适配项目的 Dark/Light 主题切换逻辑
