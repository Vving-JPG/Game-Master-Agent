## Step 1.1: 修复 `abandoned` 状态不被 QuestRepo 接受

**问题**: BUG-023 — `tools.py` 中 `update_quest_status` 新增了 `"abandoned"` 状态，但 `QuestRepo.update_status` 的白名单不包含该状态，导致数据库更新静默失败，AI 被错误告知操作成功。

**涉及文件**:
- `2workbench/feature/ai/tools.py` L343
- `2workbench/core/models/repository.py` L342-344

**修改方案**（推荐方案 A + B 同时执行）：

### 方案 A：更新 `repository.py` 白名单

```python
# 文件: 2workbench/core/models/repository.py L342
# 修改前:
if status not in ("active", "completed", "failed", "not_started"):
    return False

# 修改后:
if status not in ("active", "completed", "failed", "not_started", "abandoned"):
    return False
```

### 方案 B：`tools.py` 检查返回值

```python
# 文件: 2workbench/feature/ai/tools.py L355-367
# 修改前（两处相同模式）:
quest_repo.update_status(quest.id, status)
return f"任务 [{quest_title}] 状态已更新为: {status}"

# 修改后:
success = quest_repo.update_status(quest.id, status)
if success:
    return f"任务 [{quest_title}] 状态已更新为: {status}"
else:
    return f"错误：无法将任务 [{quest_title}] 状态更新为 {status}"
```

**验收标准**:
- [ ] `update_quest_status(quest, "abandoned")` 能正确更新数据库
- [ ] 不合法的状态返回错误信息而非假成功
- [ ] AI 工具返回值与数据库实际状态一致
