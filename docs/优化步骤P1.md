## Step 1.2: 修复 event_monitor.py Monkey-patching 问题

**问题**: BUG-003 — 全局副作用、多层嵌套、无卸载机制

**文件**: `2workbench/presentation/ops/debugger/event_monitor.py` 第 71-78 行

**修改方案**:

```python
# 修改前
original_emit = event_bus.emit
def hooked_emit(event):
    ...
    return original_emit(event)
event_bus.emit = hooked_emit

# 修改后：添加卸载机制
def _setup_hooks(self) -> None:
    self._original_emit = event_bus.emit
    event_bus.emit = self._hooked_emit

def _hooked_emit(self, event) -> list:
    self._add_event_to_list(event)
    return self._original_emit(event)

def closeEvent(self, event) -> None:
    """关闭时恢复原始 emit"""
    if hasattr(self, '_original_emit'):
        event_bus.emit = self._original_emit
    super().closeEvent(event)
```

**验收标准**:
- [ ] 创建多个 EventMonitor 实例不会导致嵌套调用
- [ ] 关闭 EventMonitor 后钩子被正确移除
- [ ] 事件仍然能正常传递

## Step 1.4: 修复 graph_editor.py 边删除残留

**问题**: BUG-006（部分修复） — 边从列表移除但未从 QGraphicsScene 移除

**文件**: `2workbench/presentation/editor/graph_editor.py` 第 318-330 行

**修改方案**:

```python
# 修改前
def remove_node(self, node_id: str) -> None:
    node = self._nodes.pop(node_id, None)
    if node:
        self._edges = [
            e for e in self._edges
            if e.source.node_id != node_id and e.target.node_id != node_id
        ]
        for edge in self._edges[:]:
            if edge.scene():
                pass  # ← 空操作！
        self.removeItem(node)

# 修改后
def remove_node(self, node_id: str) -> None:
    node = self._nodes.pop(node_id, None)
    if node:
        # 先从场景中移除关联的边
        for edge in self._edges[:]:
            if edge.source.node_id == node_id or edge.target.node_id == node_id:
                self.removeItem(edge)
        # 过滤边列表
        self._edges = [
            e for e in self._edges
            if e.source.node_id != node_id and e.target.node_id != node_id
        ]
        # 移除节点
        self.removeItem(node)
```

**验收标准**:
- [ ] 删除节点后，关联的边从场景中消失
- [ ] 无内存泄漏（边对象被正确清理）

## Step 1.6: 修复 safety_panel.py 安全级别逻辑

**问题**: BUG-008 — STANDARD 和 RELAXED 对 strict 规则无区别

**文件**: `2workbench/presentation/ops/safety/safety_panel.py` 第 259-261 行

**修改方案**:

```python
# 修改后：明确三个级别的行为
for rule in self._rules:
    if rule.level == "strict":
        if level == SafetyLevel.RELAXED:
            continue  # RELAXED: 跳过 strict
        # STANDARD 和 STRICT: 都应用 strict 规则
    elif rule.level == "standard":
        if level == SafetyLevel.RELAXED:
            continue  # RELAXED: 也跳过 standard
    # relaxed 规则在所有级别都应用
    rule.enabled = True
```

**验收标准**:
- [ ] RELAXED 模式：只应用 relaxed 规则
- [ ] STANDARD 模式：应用 relaxed + standard 规则
- [ ] STRICT 模式：应用所有规则
