## Step 0.1: 修复 `_on_agent_error` 方法名冲突

**问题**: BUG-024 — `_on_agent_error` 在 main_window.py 中被定义了两次（L901 和 L1341），Python 后定义覆盖前定义，导致 EventBus 回调接收到错误的参数类型。

**文件**: `2workbench/presentation/main_window.py`

**修改方案**：

1. 将 L901 的 EventBus 回调版本改名为 `_on_eventbus_agent_error`：
```python
# L901 修改前:
def _on_agent_error(self, event: Event) -> None:
    error = event.get("error", "未知错误")
    ...

# L901 修改后:
def _on_eventbus_agent_error(self, event: Event) -> None:
    error = event.get("error", "未知错误")
    ...
```

2. 将 L883 的订阅改为新方法名：
```python
# L883 修改前:
event_bus.subscribe("feature.ai.agent_error", self._on_agent_error)

# L883 修改后:
event_bus.subscribe("feature.ai.agent_error", self._on_eventbus_agent_error)
```

3. L1341 的 `_on_agent_error(self, error: str)` 保持不变（AgentThread.error 信号回调）

**验收标准**:
- [ ] EventBus 发出 agent_error 事件时，工具栏正确显示错误状态
- [ ] AgentThread.error 信号触发时，正确显示错误弹窗
- [ ] 两个回调互不干扰

## Step 0.4: 修复 `abandoned` 状态静默失败

**问题**: BUG-023 — `tools.py` 新增了 `abandoned` 状态，但 `repository.py` 的白名单不包含它。

**文件**: `2workbench/core/models/repository.py` L342 + `2workbench/feature/ai/tools.py` L355-367

**修改方案**：

A. `repository.py` 白名单增加 `"abandoned"`：
```python
# L342 修改前:
if status not in ("active", "completed", "failed", "not_started"):

# L342 修改后:
if status not in ("active", "completed", "failed", "not_started", "abandoned"):
```

B. `tools.py` 检查返回值（两处）：
```python
# L355-367 修改前:
quest_repo.update_status(quest.id, status)
return f"任务 [{quest_title}] 状态已更新为: {status}"

# 修改后:
success = quest_repo.update_status(quest.id, status)
if success:
    return f"任务 [{quest_title}] 状态已更新为: {status}"
else:
    return f"错误：无法将任务 [{quest_title}] 状态更新为 {status}"
```

**验收标准**:
- [ ] `update_quest_status(quest, "abandoned")` 正确更新数据库
- [ ] 不合法状态返回错误信息
