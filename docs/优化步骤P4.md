## Step 4.1: 统一日志系统 — print 替换为 logger

**问题**: QUAL-001 — 表现层大量使用 print

**涉及文件**:
- `main_window.py`（20+ 处）
- `server.py`（4 处）

**修改方案**:

逐文件替换：

```python
# 修改前
print(f"[LeftPanel] UI 初始化完成")
print(f"[MainWindow] 双击打开文件: {file_path}")

# 修改后
logger.debug(f"UI 初始化完成")
logger.info(f"双击打开文件: {file_path}")
logger.warning(f"项目路径为空")
logger.error(f"添加树节点错误: {e}")
```

**日志级别使用规范**:
| 级别 | 用途 |
|------|------|
| `DEBUG` | UI 初始化、事件触发、数据加载细节 |
| `INFO` | 项目打开/关闭、文件操作、用户操作 |
| `WARNING` | 非致命异常、配置缺失、降级处理 |
| `ERROR` | 操作失败、异常捕获 |

**验收标准**:
- [ ] `main_window.py` 中无 `print()` 调用
- [ ] `server.py` 中无 `print()` 调用
- [ ] 日志级别使用合理

## Step 4.4: 修复 server.py 线程安全问题

**问题**: QUAL-008 — 类变量无同步机制

**文件**: `2workbench/presentation/server.py`

**修改方案**:

```python
import threading

class GUIControlHandler(http.server.BaseHTTPRequestHandler):
    gui_instance = None
    project_root = ""
    _lock = threading.Lock()

    @classmethod
    def set_gui(cls, gui, root):
        with cls._lock:
            cls.gui_instance = gui
            cls.project_root = root

    def _get_gui(self):
        with cls._lock:
            return self.gui_instance, self.project_root
```

**验收标准**:
- [ ] HTTP 线程和主线程安全访问共享变量
