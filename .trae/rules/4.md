## 1. NPC系统 (步骤 3.1-3.7)

### 1.1 扩展NPC属性模型 (3.1)

**数据库表**: `npcs` 表新增字段
- `mood` TEXT - 当前心情 (neutral/happy/angry/confident等)
- `goals` TEXT (JSON) - 目标列表 [{"description":"...", "priority":1}]
- `relationships` TEXT (JSON) - 关系字典 {"npc_id": value, "player_id": value}
- `speech_style` TEXT - 说话风格描述

**文件**: `src/models/npc_repo.py`
- `create_npc()` 支持 mood/goals/relationships/speech_style 参数
- `get_npc()` 自动反序列化 JSON 字段
- `update_npc()` 自动序列化 dict/list 类型的 JSON 字段

### 1.2 NPC性格模板 (3.2)

**文件**: `src/data/npc_templates.py`

| 模板名 | 大五人格特征 | 说话风格 | 默认心情 |
|--------|-------------|----------|----------|
| brave_warrior | 高尽责/高外向/低神经质 | 直率豪爽！ | confident |
| mysterious_mage | 高开放/低外向 | 隐晦深奥... | contemplative |
| friendly_merchant | 高外向/高宜人性 | 热情亲切 | cheerful |
| sinister_villain | 高尽责/低宜人性 | 阴阳怪气 | cunning |
| wise_elder | 高开放/高尽责 | 缓慢庄重 | serene |
| naive_villager | 高宜人性/高神经质 | 朴实紧张 | curious |

**函数**:
- `get_template(name)` - 获取模板
- `list_templates()` - 列出所有模板
- `apply_template(name, npc_name, overrides)` - 应用模板生成属性

### 1.3 create_npc工具 (3.3)

**文件**: `src/tools/npc_tool.py`, `src/tools/tool_definitions.py`

**工具Schema**:
```python
{
    "name": "create_npc",
    "parameters": {
        "name": str,              # 必需
        "location_id": int,       # 必需
        "personality_type": str,  # 可选: brave_warrior/mysterious_mage/...
        "mood": str,              # 可选，覆盖模板
        "speech_style": str,      # 可选，覆盖模板
        "backstory": str,         # 可选
    }
}
```

### 1.4 npc_dialog工具 (3.4)

**文件**: `src/services/npc_dialog.py`, `src/tools/npc_tool.py`

**System Prompt模板**:
```
你是一个RPG游戏中的NPC。请根据以下人设信息，用中文回复玩家。

## NPC信息
- 名字: {name}
- 心情: {mood}
- 说话风格: {speech_style}
- 性格: {personality_desc}
- 目标: {goals_desc}
- 背景故事: {backstory}
- 与玩家的关系: {relationship_desc}

## 规则
1. 严格保持角色人设
2. 根据心情调整语气
3. 回复长度1-3句话
4. 可以透露任务线索
```

**函数**: `generate_npc_dialog(npc_id, player_message, llm, db_path)`

### 1.5 NPC关系追踪 (3.5)

**工具**:
- `update_relationship(npc_id, target_id, change)` - 更新关系值
- `get_relationship_graph(npc_id=None)` - 获取关系网络

**关系值范围**: -100 ~ +100
- >50: 友好 ❤️
- 0~50: 中立
- -50~0: 冷淡
- <-50: 敌对 💔

### 1.6 NPC记忆系统 (3.6)

**文件**: `src/models/memory_repo.py`

**数据库表**: `npc_memories`
```sql
CREATE TABLE npc_memories (
    id INTEGER PRIMARY KEY,
    npc_id INTEGER,
    content TEXT,
    importance INTEGER,  -- 1-10
    created_at TIMESTAMP
);
```

**函数**:
- `add_memory(npc_id, content, importance)` - 添加记忆
- `get_memories(npc_id, limit)` - 获取记忆（按重要性排序）
- `get_important_memories(npc_id, min_importance)` - 获取重要记忆
- `compress_memories(npc_id, keep_count)` - 压缩记忆（保留最重要的N条）

### 1.7 MUD界面NPC功能 (3.7)

**文件**: `src/web/js/app.js`, `src/web/css/style.css`

**功能**:
- `loadNPCs(location_id)` - 加载地点NPC列表
- `talkToNPC(npc_id, npc_name)` - 点击NPC开始对话
- NPC面板显示：名字 + 心情标签

## 2. 剧情任务系统 (步骤 3.8-3.13)

### 2.1 扩展剧情数据模型 (3.8)

**数据库表**: `quests` 新增字段
- `prerequisites` TEXT (JSON) - 前置条件 [{"type":"quest_completed", "quest_id":1}]
- `branches` TEXT (JSON) - 分支选项 [{"id":"help", "text":"帮助", "next_step":3}]

**数据库表**: `quest_steps` 新增字段
- `completed` INTEGER DEFAULT 0 - 是否完成

**文件**: `src/models/quest_repo.py`
- `create_quest()` 支持 prerequisites/branches
- `create_quest_step()` - 创建任务步骤
- `get_quest_steps()` - 获取所有步骤
- `update_quest_step()` - 更新步骤进度
- `assign_quest()` - 分配任务给玩家

### 2.2 剧情模板系统 (3.9)

**文件**: `src/data/story_templates.py`

| 模板名 | 描述 | 步骤数 | 奖励 |
|--------|------|--------|------|
| rescue | 救援任务 | 3 | exp:150, gold:80 |
| escort | 护送任务 | 3 | exp:120, gold:60 |
| collect | 收集任务 | 2 | exp:80, gold:40 |
| investigate | 调查任务 | 4 | exp:200, gold:100 |
| exterminate | 消灭任务 | 2 | exp:100, gold:50 |

**函数**: `generate_quest_from_template(template_name, variables, quest_type)`

### 2.3 任务工具集 (3.10)

**文件**: `src/tools/quest_tool.py`

**工具**:
- `create_quest(title, description, quest_type, template_name, template_vars)`
- `update_quest_progress(quest_id, step_index, progress)`
- `handle_choice(quest_id, choice_id)`

### 2.4 剧情连贯性管理 (3.11)

**文件**: `src/services/story_coherence.py`

**函数**:
- `check_prerequisites(quest_id, player_id)` - 检查前置条件
  - 支持类型: quest_completed, level, npc_relationship
- `validate_choice(quest_id, choice_id)` - 验证分支选择
- `get_story_summary(world_id, player_id)` - 获取剧情摘要
- `format_story_context()` - 格式化剧情上下文用于GM Prompt

### 2.5 多结局系统 (3.12)

**文件**: `src/services/ending_system.py`

**结局类型**:
| 类型 | 名称 | 条件 |
|------|------|------|
| hero | 英雄结局 | 完成主线任务 + 高声望 |
| villain | 反派结局 | 背叛盟友 + 低道德 |
| neutral | 中立结局 | 平衡的选择 |
| tragic | 悲剧结局 | 自我牺牲 |
| secret | 隐藏结局 | 发现所有秘密 |

**函数**:
- `calculate_ending_score()` - 计算各结局分数
- `determine_ending()` - 确定最终结局
- `format_ending_narrative()` - 格式化结局叙事

### 2.6 MUD界面剧情功能 (3.13)

**功能**:
- 任务面板显示：标题 + 进度条
- 状态标签：active(蓝色边框) / completed(绿色边框)

## 3. 道具战斗系统 (步骤 3.14-3.18)

### 3.1 扩展道具数据模型 (3.14)

**数据库表**: `items` 已有字段
- `rarity` TEXT - common/uncommon/rare/epic/legendary
- `slot` TEXT - weapon/head/body/shield/accessory
- `stats` TEXT (JSON) - {"attack":10, "defense":5, "hp_restore":20}
- `level_req` INTEGER - 等级要求
- `stackable` BOOLEAN - 是否可堆叠
- `usable` BOOLEAN - 是否可使用

### 3.2 道具工具集 (3.15)

**文件**: `src/tools/item_tool.py`

**工具**:
- `create_item(name, item_type, rarity, stats, description, level_req, stackable, usable, slot)`
- `equip_item(item_id)` - 装备道具
- `use_item(item_id)` - 使用道具（恢复HP/MP）
- `get_inventory()` - 查看背包

### 3.3 战斗系统 (3.16)

**文件**: `src/services/combat.py`

**D&D 5e 简化规则**:
```
攻击检定: d20 + 攻击加值
命中条件: 结果 >= 目标AC
暴击: 掷出20
伤害: 武器骰 + (暴击时双倍)
```

**类**:
- `Combatant` - 战斗参与者 (name, hp, max_hp, attack_bonus, damage_dice, ac)
- `CombatResult` - 战斗结果 (attacker, defender, hit, critical, damage, narrative)

**函数**:
- `attack(attacker, defender)` - 执行攻击
- `combat_round(player, enemies)` - 执行一轮战斗
- `is_combat_over(player, enemies)` - 检查战斗结束
- `calculate_rewards(enemies)` - 计算奖励

### 3.4 战斗叙事生成 (3.17)

**文件**: `src/services/combat_narrative.py`

**函数**:
- `generate_combat_narrative(results, llm)` - 用LLM生成战斗叙事
- `generate_battle_start_narrative(enemy_names, location)` - 战斗开始
- `generate_battle_end_narrative(victory, rewards)` - 战斗结束

### 3.5 MUD界面战斗功能 (3.18)

**功能**:
- `showCombatUI(enemies)` - 显示战斗覆盖层
- `hideCombatUI()` - 隐藏战斗界面
- 敌人血条显示
- 战斗按钮：攻击 / 防御 / 逃跑

## 6. 关键设计决策记录

### 已确定
- ✅ NPC性格: 大五人格模型 + 6种预设模板
- ✅ NPC对话: LLM生成，System Prompt包含完整人设
- ✅ 关系系统: -100~100数值，NPC relationships字段存储
- ✅ 记忆系统: 独立表，支持重要性排序和压缩
- ✅ 剧情模板: 5种经典RPG任务类型
- ✅ 任务步骤: 支持进度追踪和完成标记
- ✅ 分支选择: branches JSON字段存储选项
- ✅ 多结局: 5种结局类型，基于任务完成情况计算
- ✅ 战斗系统: D&D 5e简化版（d20 + 加值 vs AC）
- ✅ 战斗叙事: LLM生成生动描述
