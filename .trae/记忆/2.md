## 2. 数据模型 (步骤 1.2)

### 数据库表结构 (11个表)

```
World (世界)
  ├── id, name, setting, created_at, updated_at
  ├── 1:N Location, Player, NPC, Quest, GameLog, GameMessage

Location (地点)
  ├── id, world_id, name, description, connections(JSON)
  ├── 1:N NPC

Player (玩家)
  ├── id, world_id, name, hp, max_hp, mp, max_mp, level, exp, gold, location_id
  ├── 1:N PlayerItem, Quest

NPC (非玩家角色)
  ├── id, world_id, name, location_id, personality(JSON), backstory, mood, goals(JSON), speech_style
  ├── 1:N NpcMemory

Item (道具模板)
  ├── id, name, item_type, rarity, slot, stats(JSON), description, level_req, stackable, usable

PlayerItem (玩家物品栏)
  ├── player_id, item_id, quantity, equipped

Quest (任务)
  ├── id, world_id, player_id, title, description, quest_type, status, rewards(JSON)
  ├── 1:N QuestStep

QuestStep (任务步骤)
  ├── id, quest_id, step_order, description, step_type, target, required_count, current_count

GameLog (游戏日志)
  ├── id, world_id, event_type, content, timestamp

NpcMemory (NPC记忆)
  ├── id, npc_id, content, importance, created_at

GameMessage (对话历史)
  ├── id, world_id, role, content, timestamp
```

### 索引
- 所有外键字段 + 常用查询字段（status, timestamp等）

## 3. GM System Prompt v1 (步骤 1.3)

### Prompt 结构

```python
SYSTEM_PROMPT = GM_ROLE + WORLD_SETTING + NARRATIVE_STYLE + BEHAVIOR_RULES + TOOL_GUIDE
```

### 核心规则
1. **永不替玩家做决定** - 描述世界和后果，选择权在玩家
2. **不要剧透** - 不提前透露未探索内容
3. **保持一致性** - 记住之前的事件和对话
4. **公平公正** - 骰子结果是最终判定
5. **适时使用工具** - 查询/修改状态时调用工具
6. **回应生动** - 避免机械式回复

### 函数
- `get_system_prompt(world_context: str = "") -> str`

## 5. Repository 层 (步骤 1.5-1.11)

### 通用设计模式

每个 Repository 包含标准 CRUD 操作：
- `create_xxx(...)` → 返回 id
- `get_xxx(id)` → 返回 dict | None
- `list_xxx_by_xxx(...)` → 返回 list[dict]
- `update_xxx(id, **kwargs)` → 返回 bool
- `delete_xxx(id)` → 返回 bool

### 特殊操作

| Repository | 特殊功能 |
|------------|----------|
| world_repo | - |
| location_repo | connections JSON 自动序列化/反序列化 |
| player_repo | `get_inventory()`, `add_item()`, `remove_item()` |
| npc_repo | personality JSON 自动处理 |
| item_repo | `search_items(name_pattern)` 模糊搜索 |
| quest_repo | `update_quest_status()` 状态校验 (active/completed/failed) |
| log_repo | `VALID_EVENT_TYPES` 校验, `get_recent_logs(limit)` |

### JSON 字段处理
- 写入: `json.dumps(data, ensure_ascii=False)`
- 读取: `json.loads(raw_data)`
