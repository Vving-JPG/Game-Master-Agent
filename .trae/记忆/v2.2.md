## 1. 新增模块（6 个文件）

### 1.1 src/agent/command_parser.py

**类**: `CommandParser`

**核心方法**: `parse(raw_text: str) -> dict`

**4 级容错策略**:
1. 直接 `json.loads()` 解析
2. 提取 ` ```json ... ``` ` 代码块
3. 提取最外层 `{ ... }` JSON 对象
4. 兜底：整个文本作为 narrative

**返回格式**:
```python
{
    "narrative": str,
    "commands": [{"intent": str, "params": dict}],
    "memory_updates": [{"file": str, "action": str, "content": str}]
}
```

内置 `_normalize()` / `_normalize_commands()` / `_normalize_memory_updates()` 确保缺失字段有默认值。

### 1.2 src/agent/prompt_builder.py

**类**: `PromptBuilder`

**构造函数参数**: `system_prompt_path`, `memory_manager`, `skill_loader`

**核心方法**:
| 方法 | 功能 |
|------|------|
| `load_system_prompt()` | 加载 system_prompt.md（带缓存） |
| `invalidate_system_prompt_cache()` | 清除缓存 |
| `build(event, history, memory_depth)` | 组装完整 messages 列表 |

**build() 组装顺序**:
1. System prompt（基础 system_prompt.md）
2. 相关 Skill（通过 `skill_loader.get_relevant_skills()` 匹配，嵌入激活层内容）
3. 记忆上下文（`memory_manager.load_context()` 加载，仅当有 context_hints）
4. 游戏状态快照（`event.game_state` 键值对）
5. 历史对话（`history`）
6. 当前事件（格式化为 `[event_type] 玩家: raw_text`）

**事件格式化逻辑** (`_format_event_as_user_message`):
- 有 `raw_text` → `[event_type] 玩家: {raw_text}`
- 无 raw_text → 按事件类型映射描述（`player_move` / `combat_start` / `npc_interact` 等 9 种）

### 1.3 prompts/system_prompt.md

**文件大小**: 2589 字符

**结构**:
- **角色定义**: GM Agent 的 4 项职责
- **输出格式**: JSON `{narrative, commands, memory_updates}`
- **narrative 规范**: 中文第二人称，日常 100-200 字，重要 300-500 字
- **commands 规范**: 只发 Skill 中 allowed-tools 允许的指令
- **memory_updates 规范**: action 类型 (append/create/update_frontmatter)，记录格式 `[第X天 时间段]`
- **可用指令表**: 11 条 intent（update_npc_relationship ~ no_op）
- **Skill 使用规则**: 按事件类型/关键词匹配 → 只使用 allowed-tools
- **记忆管理规则**: 每回更新，不重复记录
- **Agent 创建 Skill 规则**: 通过 memory_updates 的 create action
- **引擎拒绝处理**: 不自动重试，记入 session/current.md

### 1.4 src/agent/event_handler.py

**类**: `EventHandler`

**构造函数参数**: `game_master`, `engine_adapter`

**核心方法**:
| 方法 | 功能 |
|------|------|
| `register_sse_callback(callback)` | 注册 SSE 推送回调 |
| `handle_event(event) -> dict` | 处理事件并推送 SSE（有并发保护） |
| `stream_response(event) -> AsyncIterable[dict]` | 流式处理，yield SSE 事件字典 |

**SSE 推送时序**: turn_start → (GameMaster 内部流式) → command → memory_update → command_rejected → turn_end

**并发保护**: `_processing` 标志位，处理中忽略新事件，返回 `{"error": "AGENT_BUSY"}`

## 2. 修改的核心模块

### 2.1 src/services/llm_client.py — 同步 → 异步

**关键变更**:
```diff
- from openai import OpenAI
+ from openai import AsyncOpenAI

- self.client = OpenAI(...)
+ self.client = AsyncOpenAI(...)

- def chat(self, ...):
+ async def chat(self, ...):

- def _call_api(self, ...):
+ async def _call_api(self, ...):

- self.client.chat.completions.create(...)
+ await self.client.chat.completions.create(...)
```

**新增 `self.model_name`**: 存储原始模型名，供 `stream()` 方法用

**新增 `_sync_chat()`**: `asyncio.run(self.chat(...))` 兼容旧代码（已废弃标记）

**新增 `stream()` 方法**:
- `async def stream(messages, model, tools, temperature, **kwargs)`
- yield 事件字典: `reasoning` / `token` / `llm_complete`
- 处理 DeepSeek `reasoning_content`（`getattr(delta, 'reasoning_content', None)`）
- 增量拼接 tool_calls（按 index 累积 `tc.function.arguments`）
- 最终 `llm_complete` 事件包含 `content`, `reasoning_content`, `tool_calls`, `finish_reason`

**变更影响的 V1 方法**:
| 方法 | 变更 |
|------|------|
| `chat()` | `def` → `async def`，加 `await` |
| `chat_with_tools()` | `def` → `async def`，加 `await`，改 `retry` 装饰 |
| `chat_stream()` | `def` → `async def`，`for` → `async for` |
| `_call_api()` | `def` → `async def`，加 `await` |

### 2.2 4 个 V1 服务文件 async 适配

| 文件 | 变更 |
|------|------|
| `src/services/pregenerator.py` | `llm.chat(...)` → `await llm.chat(...)` x2 |
| `src/services/npc_dialog.py` | `generate_npc_dialog` → `async def`，`llm.chat(...)` → `await llm.chat(...)` |
| `src/services/combat_narrative.py` | `generate_combat_narrative` → `async def`，`llm.chat(...)` → `await llm.chat(...)` |
| `src/services/info_extractor.py` | `extract_key_info` / `build_world_summary` → `async def`，内部 `await` |

### 2.3 src/agent/game_master.py — 完全重写

**旧（V1）**:
```python
class GameMaster:
    def __init__(self, world_id, player_id, llm_client, db_path):
        # SQLite 连接、加载历史、tools 列表

    def process(self, user_input: str) -> str:
        # while tool_round < MAX_TOOL_ROUNDS:
        #   response = self.llm.chat_with_tools(messages, self.tools)
        #   if tool_calls → execute_tool() → continue
        #   else → return reply
```

**新（V2）**:
```python
class GameMaster:
    def __init__(self, llm_client, memory_manager, skill_loader, engine_adapter, system_prompt_path):
        # 组装 CommandParser + PromptBuilder + MemoryManager + SkillLoader + EngineAdapter

    async def handle_event(self, event: EngineEvent) -> dict:
        # 1. PromptBuilder.build(event, history)
        # 2. llm_client.stream(messages)  # 流式调 LLM
        # 3. CommandParser.parse(full_content)
        # 4. memory_manager.apply_memory_updates()
        # 5. engine_adapter.send_commands()
        # 6. memory_manager.apply_state_changes()  # 引擎状态 → YAML
        # 7. _update_history() / _update_session()
        # return {response_id, narrative, commands, memory_updates, command_results, stats}
```

**关键设计变化**:
- 不再持有 `world_id` / `player_id` / `db_path` — 通过 TextAdapter 桥接
- 不再 `while` 循环 — 每轮独立调用 `handle_event()`
- 不再 `chat_with_tools()` — 用 `stream()` 流式获取完整响应后解析
- 新增 `_update_history()`: 保留最近 20 轮（40 条消息）
- 新增 `_update_session()`: 写入 `session/current.md`
- 新增 `reset()`: 重置 turn/history/tokens

## 6. 关键设计决策记录

### 已确定

- ✅ **LLMClient → AsyncOpenAI**: 改为异步以匹配整个 P1 async 调用链
- ✅ **GameMaster 构造函数变更**: `(world_id, player_id, llm, db_path)` → `(llm, memory_manager, skill_loader, engine_adapter, system_prompt_path)`
- ✅ **Handle Event 模式**: 单事件 → 单 handle_event() 调用，不再 while 循环
- ✅ **Stream 替代 chat_with_tools**: V2 不依赖 OpenAI function calling，由 LLM 在 content 中输出 JSON 后解析
- ✅ **4 级容错 JSON 解析**: 应对 LLM 输出格式不稳定
- ✅ **SSE 推送架构**: EventHandler 通过 callback 推送，支持 FastAPI EventSourceResponse
- ✅ **V1 服务文件同步改异步**: 4 个文件加了 await，接口签名从 `def` 改 `async def`

### 待后续确定

- ⏳ API 层 (ws.py, action.py) 如何适配新 GameMaster 接口
- ⏳ 是否需要保留旧 `chat_with_tools()` 作为降级通道
- ⏳ WebSocket / SSE 在 FastAPI 中的集成测试
