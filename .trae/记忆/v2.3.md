### 3.2 文件浏览器 (FileTree.vue)

**关键修复**: 点击文件夹时不触发文件加载

```typescript
function handleSelect(keys: string[], option: TreeOption[]) {
  selectedKeys.value = keys
  if (keys.length > 0 && option.length > 0) {
    // 只选择文件，不选择文件夹
    if (option[0].isLeaf) {
      emit('file-selected', keys[0])
    }
  }
}
```

**功能**:

## 5. 修复记录

### 5.1 文件加载失败问题

**现象**: 点击文件夹时显示"加载文件失败"

**原因**: FileTree.vue 中点击任何节点都会触发 `file-selected` 事件，后端 `/file` 端点对文件夹返回 400 Bad Request

**修复**: 修改 `handleSelect`，只在 `isLeaf=true`（文件）时触发事件

### 5.2 Naive UI 组件未注册

**现象**: 控制台显示 `Failed to resolve component: n-tree` 等错误

**修复**: 在每个组件中显式导入使用的 Naive UI 组件

```typescript
import { NInput, NTree, useMessage } from 'naive-ui'
```

### 5.3 SSE 响应头缺失

**现象**: SSE 连接建立但无法接收事件

**修复**: 后端添加 SSE 专用响应头

```python
return StreamingResponse(
    _generate_events(session_id, last_event_id),
    media_type="text/event-stream",
    headers={
        "Cache-Control": "no-cache",
        "Connection": "keep-alive",
        "X-Accel-Buffering": "no",
    },
)
```

## 9. 关键设计决策

### 9.1 为什么使用 MockAdapter
V2 架构中 TextAdapter 依赖 V1 的服务层，但 V2 尚未完全实现这些服务。使用 MockAdapter 可以让前端开发独立于后端数据层完成。

### 9.2 为什么显式导入 Naive UI 组件
Vue 3 的 `<script setup>` 不会自动注册组件，必须显式导入。这是 TypeScript 严格模式的要求。

### 9.3 为什么 SSE 需要特殊 headers
SSE 流式响应需要禁用缓存和缓冲，否则浏览器或代理服务器可能会缓冲响应，导致事件无法实时到达客户端。
