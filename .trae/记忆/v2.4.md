## 2. 关键变更

### 2.1 清理 V1 遗留代码

**问题**: `ws.py` 和 `action.py` 仍使用 V1 旧构造函数 `GameMaster(world_id, player_id, llm_client)`，与 V2 新接口不兼容。

**解决方案**:
- 将 WebSocket 和 Action 端点标记为 **deprecated**
- 返回 410 Gone 状态码，提示用户使用 V2 端点
- 保留旧代码注释作为迁移参考

```python
# src/api/routes/ws.py
@router.websocket("/ws/worlds/{world_id}")
async def game_websocket(websocket: WebSocket, world_id: int):
    """游戏 WebSocket 连接 (已弃用 - V2 使用 SSE 替代)"""
    await websocket.send_json({
        "type": "system",
        "content": "WebSocket 端点已弃用。请使用 /api/agent/events SSE 端点。"
    })
    await websocket.close()
```

```python
# src/api/routes/action.py
@router.post("/action")
def game_action(world_id: int, body: ActionRequest):
    """玩家行动 → GM回复 (已弃用 - V2 使用 /api/agent/event 替代)"""
    raise HTTPException(
        status_code=410,  # Gone
        detail="此端点已弃用。请使用 /api/agent/event 端点。"
    )
```

### 2.2 TextAdapter 命令行模式

**新文件**: `src/cli_v2.py`

功能:
- 通过 TextAdapter 连接 SQLite 引擎
- 启动事件驱动的 Agent 循环
- 支持 `quit`, `status`, `help` 命令
- 显示回合开始/结束事件和 token 消耗

```bash
uv run python src/cli_v2.py
```

核心流程:
1. 初始化 LLMClient
2. 初始化 MemoryManager 和 SkillLoader
3. 连接 TextAdapter
4. 创建 GameMaster 和 EventHandler
5. 注册 SSE 回调打印关键事件
6. 进入交互循环

## 6. 关键设计决策

### 6.1 为什么弃用而不是删除 ws.py/action.py?

- 保留 WebSocket 框架代码作为迁移参考
- 返回明确的 410 Gone 状态码，提示用户使用新端点
- 避免破坏现有客户端的意外行为（如 500 错误）

### 6.2 性能测试使用 mock LLM

- 测量的是框架开销而非 LLM 延迟
- 确保测试结果稳定可重复
- 实际 LLM 延迟取决于网络和服务商

### 6.3 集成测试放在 tests/test_integration/

- 与单元测试分离，便于分类运行
- 集成测试可能更慢，可单独执行
- 清晰的测试层次结构
