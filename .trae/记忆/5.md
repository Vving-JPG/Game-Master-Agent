## 1. API 设计 (步骤 4.1)

**文件**: `docs/api_design.md`

### REST 端点

| 类别 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 世界 | GET/POST/DELETE | /api/worlds | 世界 CRUD |
| 玩家 | GET/PATCH | /api/worlds/{id}/player | 玩家信息/更新 |

- 端点: `ws://localhost:8000/ws/worlds/{world_id}`
- 客户端→服务端: `{"type": "action", "content": "玩家输入"}`
- 服务端→客户端: `{"type": "narrative", "content": "文本"}` / `{"type": "system"}` / `{"type": "combat"}` / `{"type": "choice"}`

## 6. 前端升级 (步骤 4.9-4.11)

### 真实 WebSocket 连接 (`src/web/js/app.js`)
```javascript
class RealGameClient {
    constructor(worldId) {
        this.ws = new WebSocket(`ws://${host}/ws/worlds/${worldId}`);
    }
    
    connect() {
        this.ws.onopen = () => { ... }
        this.ws.onmessage = (event) => {
            const data = JSON.parse(event.data);
            if (data.type === 'narrative') appendToLastGMMessage(data.content);
            else if (data.type === 'system') addMessage(data.content, 'system-text');
            else if (data.type === 'combat') handleCombatUpdate(data.data);
            else if (data.type === 'choice') showChoices(data.choices);
        }
        this.ws.onclose = () => { /* 5秒后重连 */ }
    }
}
```

## 7. 管理端：Prompt 管理系统 (步骤 4.15)

### 数据库表
```sql
CREATE TABLE prompt_versions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    prompt_key TEXT NOT NULL,      -- "game_master_system"
    content TEXT NOT NULL,         -- Prompt 内容
    version INTEGER NOT NULL,      -- 版本号
    is_active INTEGER DEFAULT 0,   -- 是否活跃
    created_at TEXT
);
```

### 数据访问 (`src/models/prompt_repo.py`)
- `save_prompt(prompt_key, content)` - 保存新版本，自动递增版本号
- `get_active_prompt(prompt_key)` - 获取当前活跃版本
- `get_prompt_history(prompt_key, limit)` - 获取版本历史
- `rollback_prompt(prompt_key, version)` - 回滚到指定版本

### API 路由 (`src/api/routes/admin_prompts.py`)
- `GET /api/admin/prompts/{prompt_key}` - 获取当前 Prompt
- `POST /api/admin/prompts` - 更新 Prompt（立即生效）
- `GET /api/admin/prompts/{prompt_key}/history` - 版本历史
- `POST /api/admin/prompts/rollback` - 回滚版本

## 8. 管理端：AI 行为监控 (步骤 4.16)

### 数据库表
```sql
CREATE TABLE llm_calls (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    world_id INTEGER,
    call_type TEXT DEFAULT 'chat',        -- chat/chat_with_tools
    prompt_tokens INTEGER DEFAULT 0,
    completion_tokens INTEGER DEFAULT 0,
    total_tokens INTEGER DEFAULT 0,
    latency_ms INTEGER DEFAULT 0,         -- 延迟
    tool_calls_count INTEGER DEFAULT 0,
    tool_names TEXT DEFAULT '[]',         -- 调用的工具列表
    error TEXT DEFAULT '',
    created_at TEXT
);
```

### 数据访问 (`src/models/metrics_repo.py`)
- `record_llm_call(...)` - 记录每次 LLM 调用
- `get_recent_calls(world_id, limit)` - 获取最近调用记录
- `get_token_stats()` - 获取 Token 统计（总调用次数、总 Token、平均延迟、错误数）

### API 路由 (`src/api/routes/admin_monitor.py`)
- `GET /api/admin/monitor/calls` - 最近调用记录

## 14. 关键设计决策记录

### 已确定
- ✅ Web 框架: FastAPI + Uvicorn
- ✅ 实时通信: WebSocket (原生，非 Socket.IO)
- ✅ 认证: API Key Header (开发环境可选)
- ✅ 错误处理: 统一 JSON 格式 + 自定义异常类
- ✅ 连接管理: 内存字典存储 (world_id -> WebSocket 集合)
- ✅ Prompt 管理: 版本化存储，支持热更新
- ✅ AI 监控: 记录每次 LLM 调用的 Token 和延迟
- ✅ 管理端前端: Vue 3 + Naive UI

### 待 P5 确定
- ⏳ 生产环境部署 (Docker)
- ⏳ 性能优化 (缓存、连接池)
- ⏳ 多世界并发支持
- ⏳ 更复杂的权限系统
