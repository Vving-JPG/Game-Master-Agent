## 项目概述

V1 做了一个 MUD 文字游戏，V2 要做一个**通用游戏驱动 Agent 服务**——类似 Trae 驱动代码，Agent 驱动游戏。Agent 不再是游戏本身，而是通过标准化协议与任意游戏引擎通信的独立服务。

- **技术栈**: Python 3.11+ / DeepSeek API / FastAPI / SQLite / python-frontmatter
- **包管理器**: uv
- **LLM**: DeepSeek（通过 OpenAI 兼容接口调用）
- **开发IDE**: Trae

## 5. Skill 系统 (步骤 0.7)

### 5.1 loader.py - Skill 发现与加载器

**文件**: `src/skills/loader.py`

**数据类**: `SkillMetadata`

```python
@dataclass
class SkillMetadata:
    name: str
    description: str
    version: str
    tags: list[str]
    triggers: list[dict]      # event_type / keyword / memory_hint
    file_path: str
    source: str               # builtin / agent_created
```

**类**: `SkillLoader`

**核心方法**:

| 方法 | 功能 |
|------|------|
| `discover_all()` | 扫描 builtin/ + agent_created/，返回带缓存的 Skill 列表 |
| `get_relevant_skills(event_type, user_input, context_hints)` | 按相关度评分排序 |
| `load_skill_content(skill_name)` | 加载完整 Markdown Body |
| `load_skill_activation(skill_name)` | 加载激活层（YAML + 前2000字符） |
| `invalidate_cache()` | 清除缓存 |

**相关度评分规则**:
- event_type 匹配: +10
- keyword 匹配: +5/关键词
- memory_hint 匹配: +3/hint

## 6. 引擎适配层 (步骤 0.8-0.9)

### 6.1 base.py - 抽象接口 (0.8)

**文件**: `src/adapters/base.py`

**核心数据类**:

```python
@dataclass
class EngineEvent:       # 引擎 → Agent
    event_id: str
    timestamp: str
    type: str            # player_action / player_move / combat_start
    data: dict           # {raw_text, player_id}
    context_hints: list[str]
    game_state: dict     # {location, player_hp, player_level}

@dataclass
class CommandResult:     # 指令执行结果
    intent: str
    status: str          # success / rejected / partial / error
    new_value: any
    state_changes: dict  # {file, frontmatter}
    reason: str
    suggestion: str
```

**抽象基类**: `EngineAdapter` (ABC)

| 抽象方法 | 功能 |
|----------|------|
| `name` | 适配器名称 |
| `connection_status` | 连接状态 (DISCONNECTED/CONNECTING/CONNECTED/ERROR) |
| `connect(**kwargs)` | 连接引擎 |
| `disconnect()` | 断开连接 |
| `send_commands(commands)` | 发送指令列表 → 返回结果列表 |
| `subscribe_events(event_types, callback)` | 订阅事件 |
| `query_state(query)` | 查询游戏状态 |
| `health_check()` | 健康检查（非抽象，有默认实现） |

### 6.2 text_adapter.py - MUD 文字游戏适配器 (0.9)

**文件**: `src/adapters/text_adapter.py`

**类**: `TextAdapter(EngineAdapter)`

**支持的 Intent** (10种):

| Intent | 功能 |
|--------|------|
| `update_npc_relationship` | 更新 NPC 好感度 |
| `update_npc_state` | 更新 NPC 状态字段 |
| `offer_quest` | 发布任务 |
| `update_quest` | 更新任务进度/状态 |
| `give_item` | 给予物品 |
| `remove_item` | 移除物品 |
| `modify_stat` | 修改玩家属性 |
| `teleport_player` | 传送玩家 |
| `show_notification` | 显示通知 |
| `no_op` / `play_sound` | 空操作 |

## 13. V1 经验教训（已遵守）

1. **PowerShell `&&` 语法**: Windows PowerShell 不支持 `&&`，用 `;` 分隔多条命令
2. **测试隔离**: 每个测试模块用 `teardown_module()` 清理全局状态，防止测试间污染
3. **SQLite datetime('now')**: 同一秒内多次调用返回相同时间戳，测试断言用 `>=` 而非 `==`
4. **中文括号**: 测试代码中一律用英文括号 `()`，不要用中文括号 `（）`
5. **原子写入**: 所有 .md 文件写入必须用 `atomic_write()`，不要直接 `open().write()`
6. **YAML Front Matter 格式**: 用 `python-frontmatter` 库解析，不要手写字符串拼接
