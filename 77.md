# WorldSim-Master 项目完整架构分析

## 一、项目定位

这是一个 **AI 驱动的通用游戏 Master Agent 服务**（V2 架构）。核心理念：**Agent 不是游戏本身，而是驱动游戏的独立服务层**，通过标准化 JSON 协议与任意游戏引擎通信。

## 三、逐文件详细分析

### 3.1 Agent 核心模块 (`src/agent/`)

#### `game_master.py` (338 行)
**作用**: Game Master Agent 的主类，事件驱动的核心处理器

**核心功能**:
- `handle_event()` - 处理单个引擎事件的完整流程（8步）
- `_update_history()` - 维护最近20轮对话历史
- `_update_session()` - 更新会话记录文件
- `_register_workflow_handlers()` - 注册工作流步骤处理器
- `pause/resume/step_once` - 工作流控制（暂停/恢复/单步）

**处理流程**:
1. 组装 Prompt → 2. 流式调用 LLM → 3. 解析输出 → 4. 更新记忆 → 5. 发送指令 → 6. 更新历史

#### `command_parser.py` (130 行)
**作用**: LLM 输出解析器，将文本转为标准 JSON 命令流

**容错策略**（4层）:
1. 直接 JSON 解析
2. 提取 ` ```json ... ``` ` 代码块
3. 提取最外层 `{ ... }`
4. 兜底：整个文本作为 narrative

**输出格式**:
```json
{
  "narrative": "叙事文本",
  "commands": [{"intent": "...", "params": {}}],
  "memory_updates": [{"file": "...", "action": "append", "content": "..."}]
}
```

#### `prompt_builder.py` (142 行)
**作用**: Prompt 组装器，整合 system prompt + Skill + 记忆 + 事件

**组装内容**:
1. 基础 system prompt（从文件加载，带缓存）
2. 相关 Skill（通过 trigger 匹配）
3. 记忆上下文（渐进式加载）
4. 游戏状态快照
5. 当前事件（格式化为 user 消息）

#### `event_handler.py` (196 行)
**作用**: 事件分发与 SSE 推送处理器

**SSE 事件时序**:
1. `turn_start` - 回合开始
2. `reasoning/token` - 流式叙事（由 GameMaster 内部推送）
3. `command` - 单条指令
4. `memory_update` - 记忆更新
5. `command_rejected` - 指令被拒绝
6. `turn_end` - 回合结束

#### `workflow.py` (236 行)
**作用**: 工作流引擎，支持 YAML 定义的可暂停执行流程

**步骤类型**:
- `PROMPT` - 组装 Prompt
- `LLM_STREAM` - 流式调用 LLM
- `PARSE` - 解析输出
- `BRANCH` - 条件分支
- `EXECUTE` - 执行指令
- `MEMORY` - 更新记忆
- `END` - 结束

**控制方法**: `pause()`, `resume()`, `step_once()`

### 3.2 记忆系统 (`src/memory/`)

#### `manager.py` (61 行)
**作用**: 记忆管理器主类，整合文件读写和渐进式加载

**核心方法**:
- `load_context()` - 根据 depth 加载不同层级记忆
- `load_full_file()` - 加载完整文件（YAML + Markdown）
- `apply_state_changes()` - 引擎执行后更新 YAML Front Matter
- `apply_memory_updates()` - Agent 每回合追加 Markdown Body
- `initialize_workspace()` - 初始化 workspace 目录结构

#### `loader.py` (64 行)
**作用**: 渐进式记忆加载器（3层披露）

**加载层级**:
| 层级 | 方法 | 内容 | Token 估算 |
|------|------|------|-----------|
| Layer 1 | `load_index()` | name, type, tags, version | ~100/file |
| Layer 2 | `load_activation()` | 完整 YAML + 章节标题 | ~500-2000/file |
| Layer 3 | `load_execution()` | 完整文件内容 | ~2000-5000/file |

#### `file_io.py` (66 行)
**作用**: 记忆文件原子读写模块

**核心函数**:
- `atomic_write()` - 原子写入（tempfile + os.replace）
- `update_memory_file()` - 统一更新接口（自动递增 version）
