## 可能原因（待验证）

1. **Naive UI CSS-in-JS 注入失败** — Naive UI 在运行时通过 JavaScript 注入 CSS 变量，可能因为某些原因未执行。`n-config-provider` 的 darkTheme 传递使用了 `:theme="gt(P_)"` 的编译形式。

2. **模块加载顺序问题** — Vue SFC 编译出的 `<style>` 块可能覆盖或与 Naive UI 的内部样式冲突。

3. **StaticFiles mount 路径问题** — FastAPI 将 `/admin` mount 到 `dist` 目录，`html=True` 参数可能有边界情况。

4. **浏览器兼容性** — 预览工具内嵌的 WebView 可能不支持某些 ES module 特性。

5. **Vue 组件异步加载失败** — `currentComponent` 使用 `markRaw` + dynamic `component :is`，如果子组件导入失败会静默空白。

6. **`n-config-provider` 的 `inline-theme-disabled`** — Naive UI 2.44 版本可能有主题注入的已知问题。

## 建议下一步

1. 在真实浏览器（Chrome/Firefox）而非 IDE 预览中打开 `http://localhost:8000/admin/simple.html`
2. 打开 `http://localhost:8000/admin/vuetest.html` 看 CDN 版本是否正常
3. 打开 `http://localhost:8000/admin` 并查看 F12 Console / Network 面板
4. 如果 simple.html 也不显示，问题在 StaticFiles 层面
5. 如果 vuetest.html 正常但 admin 不正常，问题在本地的 Naive UI bundle
