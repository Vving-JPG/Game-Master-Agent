## 一、删除旧代码和冗余文件

### 1.1 删除 _legacy 目录

```bash
rm -rf 2workbench/_legacy/
```

**说明**：`_legacy/` 目录包含约40个旧版文件，已被新四层架构完全替代。保留它只会增加维护负担和混淆。

**确认清单**：
- [ ] `grep -rn "_legacy" 2workbench/ --include="*.py" | grep -v "__pycache__"` 无引用
- [ ] `grep -rn "_legacy" 2workbench/app.py 2workbench/main.py` 无引用

### 1.2 整理根目录散落文件

根目录下有大量散落的 `.md` 文件，应整合到 `docs/` 目录：


## 三、拆分 feature/ai/tools.py

**当前问题**：`tools.py` 有 835 行，包含所有工具定义，文件过大。

### 建议结构

```
feature/ai/tools/
├── __init__.py          # 导出 get_all_tools, register_tool 等
├── context.py           # ToolContext, ToolResult
├── core_tools.py        # dice_roll, check_quest_prerequisites 等
├── item_tools.py        # give_item, remove_item, use_item
├── world_tools.py       # move_to_location, update_npc_relationship
├── quest_tools.py       # create_quest, update_quest_status
└── memory_tools.py      # store_memory, recall_memory
```

### 执行步骤

```bash
mkdir -p 2workbench/feature/ai/tools

# 1. 创建 context.py — 提取 ToolContext 和 ToolResult
# 2. 按功能拆分工具函数到对应文件
# 3. 创建 __init__.py 统一导出
```

### `feature/ai/tools/__init__.py`

```python
from feature.ai.tools.context import ToolContext, ToolResult
from feature.ai.tools.core_tools import dice_roll, check_quest_prerequisites
from feature.ai.tools.item_tools import give_item, remove_item, use_item
from feature.ai.tools.world_tools import move_to_location, update_npc_relationship
from feature.ai.tools.quest_tools import create_quest, update_quest_status
from feature.ai.tools.memory_tools import store_memory, recall_memory

ALL_TOOLS = [
    dice_roll, give_item, remove_item, use_item,
    move_to_location, update_npc_relationship,
    create_quest, update_quest_status,
    store_memory, recall_memory,
    check_quest_prerequisites,
]

def get_all_tools() -> list:
    return ALL_TOOLS

def register_tool(tool):
    ALL_TOOLS.append(tool)
```
