# 工作流: 热修复流程

## 快速修复步骤

1. **定位问题**: 使用 `debug_agent.md` 中的排查方法
2. **修改代码**: 直接修改对应层的文件
3. **验证修复**: 运行对应层的测试
4. **回归测试**: 运行全量集成测试

## 注意事项

- 修改 Foundation 层可能影响所有上层，修改后需全量测试
- 修改 Core 层需验证所有 Repository 方法
- 修改 Feature 层需验证 EventBus 事件流
- 修改 Presentation 层风险最低，通常只需重启 UI

## 回滚

如果修复导致问题，可以从 `_legacy/` 目录参考原始实现。
